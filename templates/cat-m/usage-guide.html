Part VIIIA Category M file upload guidance

1. File structure

Your file must be a single‑sheet .xlsx workbook. The data table must start in the first worksheet, at cell A1, and there must be no other worksheets or data outside the table.

The table must have exactly 7 columns, in this order:

  • VMPP code
  • VMP code
  • Drug Name
  • Pack Size
  • Pack unit of measurement
  • Previous month Drug tariff price (format: Month‑YY DT, e.g. September‑24 DT)
  • Current month Drug tariff price (format: Month‑YY DT, e.g. October‑24 DT)

2. Common errors

If your upload fails, you’ll see one of these errors. Check the description and suggested fix below.

Error Meaning How to fix
Expected column “[column_name]” not found A required column header is missing or misspelt. Rename or add the header exactly as shown.
File has incorrect number of columns Your table must have exactly 7 columns. Remove extras or add missing ones so there are 7 columns.
Incorrect data type found in column “[column_name]”, row [number] A cell has the wrong format (e.g. text instead of number). Correct the data type to match expectations.
Invalid input found in column “[column_name]”, row [number] Value doesn’t match any existing database record. Check spelling/format or update the record before retrying.
An unexpected error has occurred An unexpected issue prevented upload. Try again or raise a support ticket below.
